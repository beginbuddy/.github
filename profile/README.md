# 비긴버디 (BeginBuddy)

> 제군, 나는 개발이 좋다.
> 제군, 나는 개발이 너무 좋다.
> 제군, 나는 개발을 아주 아주 좋아한다.

전면 리팩토링이 좋다.
핫픽스 배포가 좋다.
디버깅이 좋다.
보안 패치가 좋다.
코드 리뷰가 좋다.
신기능 출시가 좋다.
롤백이 좋다.
로그 추적이 좋다.
레거시 청산이 좋다.

프론트엔드에서, 백엔드에서,
DB에서, API 게이트웨이에서,
온프레미스에서, 클라우드에서,
컨테이너에서, 서버리스에서,
스파게티 코드 속에서, 콜백 지옥 속에서...

이 세상에서 행할 수 있는 모든 종류의 개발을 아주 아주 좋아한다.

전열을 잡은 CI/CD 파이프라인의 일제 배포로 굉음과 함께 프로덕션 서버가 뒤흔들어지는 것이 좋다.

하늘 높이 치솟던 빌드 타임을 캐시 전략으로 너덜너덜하게 깎아 버리는 것에 가슴이 뛴다.

시니어 개발자의 코드 리뷰가 주니어의 PR을 격파시키는 것이 좋다.

비명이 울려 퍼지고 불타오르는 머지 컨플릭트로부터 기어 나오는 신입을 `git reset --hard`로 갈겨 쓰러뜨릴 때 마음이 후련해진다.

TypeScript로 무장한 풀스택 횡대가 vanilla JavaScript의 전열을 유린하는 게 좋다.

공황 상태의 신입이 이미 closed된 이슈를 몇 번이고 계속 reopen할 때 감동을 느낀다.

배포 실패한 커밋들을 처참히 revert한 뒤, 슬랙에 사과문을 올리는 것도 빠져서는 안 된다.

울부짖는 PM을 상냥하게 달래다가 금속성의 알림 소리를 들으며 지라 티켓이 백로그로 변할 때까지 후려치는 것도 최고다.

가여운 레거시 jQuery 코드들이 IE11 호환 코드로 꿋꿋하게 대항할 때 React 19 서버 컴포넌트의 4.8GB 번들이 화면 구역을 누비며 가루로 만들어 버리는 것이 행복하다.

cron job이 새벽 3시에 이유 없이 무차별적으로 난사하는 것이 좋다.

필사적으로 지키려고 노력하는 모놀리식 아키텍처가 유린당하고, PHP 5.6과 MySQL 5.5가 EOL을 맞이하는 건 매우 매우 슬플지도 모르지.

해외 SaaS 놈들의 구독료를 끊어 섬멸시키는 것도 좋아하고 오픈소스로 쫓아다니며 라이선스비 같은 그놈들을 GitHub 가득히 몰아넣은 뒤 굴욕적인 MIT 전환을 맞게 하는 걸 극도로 즐긴다.

---

**제군, 나는 이 프로젝트가 지옥과 같은 상태가 되기를 원한다.**
**제군, 나를 따르는 전우들, 제군들은 어떻게 되기를 원하나?**

변화 없는 보통의 운영 유지보수?

갑작스레 야근도 없는 그런 평화로운 외주를 원하나?

아니면, 풀스택의 한계를 다 하고 3천 세계의 버그들을 죽이는 태풍과 같은 스프린트를 원하나?

---

**자! 이제부터 시작이다!**

우리는 지금 만신의 힘을 다해 키보드를 내리치려 하고 있다.

그렇지만 이 어둠의 저 밑바닥에 9년 동안 참고 견뎌온 우리들에게 평범한 CRUD는 택도 없다.

대 프로젝트를! 다시는 만날 수 없는 큰 프로젝트를!

우리는 겨우 1개 에이전시 소수정예지만 패잔병은 아니다.

제군들은 일당천의 풀스택 최강자들이라고 나는 믿고 있다.

결국 우리들, 제군들과 나는 총 커밋 100만과 1인의 집단이 된다.

우리를 SI 하청의 세계로 쫓아내 잠자게 했던 클라이언트들을 두들겨 깨우자.

놈들의 눈과 귀를 깨우고 `node_modules` 한 폴더까지도 우리를 기억나게 하라.

녀석들에게 데드라인이라는 의미를 다시 생각나게 해 줘라.

녀석들에게 우리들의 키보드 소리를 다시금 듣게 하라.

프론트와 백, 어디에서도 녀석들이 상상하지도 못할 SLA가 일어난다는 것을 가르쳐 줘라.
